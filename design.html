03 Module Architecture

A

Docs & Compliance

ISO document management with formal approval workflow, versioning, QR codes per published revision, category/standard grouping.

B

Production Scheduling

Shift-based calendar, worker/job/station assignments, drag-and-drop rescheduling, daily goals (per-worker, per-station, plant-wide), progress tracking.

C

Inventory & Warehouse

Part master data, stock-on-hand, transactions, BOM, 2D warehouse map view, location hierarchy browser, bin QR codes, low-stock alerts.

D

Weigh Station & Packing

Scale-integrated (RS-232/USB HID) weighing stations with USB camera, Zebra label printer, NFC tag scanning for boxing/shipment tracking, offline queue with auto-sync, all recorded per work order.

04 Document Lifecycle

Draft — Author writing
In Review — Reviewers comment
Approved — Approver signs off
Published — Frozen + QR issued
Archived — Superseded

Rejection path

If a Reviewer or Approver rejects a document, it returns to Draft with comments. Author revises and re-submits.

05 Production Scheduling & Daily Goals

Data Model

  • Shift
    name / date / start_time / end_time / station_id
  • WorkOrder
    order_number / part_id / quantity / due_date / status
    status: queued → scheduled → in_progress → complete → verified
  • Assignment
    shift / work_order / worker (User) / station / start_time / end_time / actual_start / actual_end / notes
  • WorkStation
    name / code / department / station_type (production | weigh | packing)
  • DailyGoal
    date / station_id / worker_id (optional — nil = station-level) / target_quantity / unit / note

Calendar UX

  • • Monthly / weekly / daily toggle via simple_calendar
  • • Colour-coded shift blocks on the calendar
  • • Drag-and-drop reschedule (Stimulus + SortableJS)
  • • Click a shift to see workers + work orders assigned
  • • Workers view "My Schedule" — personal shift/week view
  • • Gantt-style overview of work orders by station

Daily Goals & Progress

  • • Goals set per-station and optionally per-worker per shift
  • • Real-time progress bar on station screen: "34 / 100 units"
  • • Plant-wide rollup on the manager dashboard
  • • Auto-incremented as weigh station sessions complete
  • • Visual indicators: green (on track), yellow (behind), red (critical)

QR / NFC Job Flow

Each Assignment has a QR code. Workers scan it at the station to start work. When complete, the item moves to the weigh station — the weigh session captures weight, photo, and NFC tag on the box, then prints a label. The Assignment auto-advances and the DailyGoal increments.

06 Weigh Station & Packing

Data Model

WeighSession
work_order / assignment / part / worker (User) / weigh_station
weight_value (decimal) / unit (kg, lb, g)
photo (Active Storage — one attached image)
nfc_tag (string — scanned from box/pallet)
printed_label (boolean) / printed_at
recorded_at (timestamp)
WeighStation
name / code / description
has_scale / has_camera / has_printer / has_nfc_reader (booleans)
ip_address / port (for scale integration)
NfcTag
tag_uid (unique identifier from NFC chip)
taggable (polymorphic: Shipment, Box, Pallet)
written_at / written_by (User)
Shipment / Box
shipment_number / nfc_tag_id / contents (JSON / text)
gross_weight / net_weight / weight_unit
destination / status (packing → sealed → shipped)

Station UX Flow

STEP 1

Worker scans Assignment QR or selects WorkOrder on the tablet.

STEP 2

Places finished item on the scale. Weight auto-populates via RS-232 / USB HID. The station runs on a dedicated Ubuntu box connected to the network.

STEP 3

USB camera captures a photo of the item on the scale (evidence for QC / ISO traceability). Stored to local disk on the Ubuntu station, reported back to the server.

STEP 4

Worker scans NFC tag on the box/pallet being packed. Tag links the item to the shipment.

STEP 5

Zebra printer outputs a label. Design TBD — existing label system info to follow.

STEP 6

WeighSession saved. DailyGoal increments. Worker moves on to next item.

OFFLINE

Weigh sessions queue locally on the Ubuntu box when the server is unreachable. Auto-syncs in order when connectivity returns. No data loss on WiFi dropout.

Data Model

Part

Key Features

  • • Real-time stock-on-hand per part
  • • Low-stock alerts when below reorder_point
  • • Inventory transaction journal (auditable)
  • • BOM explosion: "what parts are needed for WO-042?"
  • • Auto-deduct from inventory when weigh session completes
  • • QR code on bin locations for mobile cycle counts
  • • CSV import / export for bulk operations

2D Warehouse Map

  • • Interactive SVG/Canvas map rendered from pos_x / pos_y coordinates on StockLocation
  • • Racks and bins drawn as coloured rectangles on a grid
  • • Colour coding by stock status: green (stocked), yellow (low), red (empty/critical)
  • • Click a bin to see contents + stock level + last transaction
  • • Zoom / pan for large warehouses
  • • Toggle heatmap mode: bins coloured by stock velocity (fast/slow movers)

Location Hierarchy Browser

  • • Tree/grid navigation: Aisle A > Rack A-03 > Bin A-03-07
  • • Breadcrumb trail showing current location in the hierarchy
  • • Search by location name, code, or scanned QR
  • • Parts list at each level with stock counts
  • • Mobile-optimized for walk-around picking and putaway
  • • Toggle between map view and hierarchy view

08 Cross-Cutting Concerns

QR Codes Everywhere

Documents (per published version), Assignments (per job), Parts (per SKU), Stock Locations (per bin), Shipments (per box).

NFC for Shipments

NFC tags on boxes/pallets scanned at the weigh station to associate weighed items with shipments. Handheld readers for packing/stacking verification.

Unified Audit

PaperTrail on all core models. WeighSessions include photos as evidence. One audit trail for ISO compliance + production traceability.

Dashboard & Reporting

Chartkick-powered live dashboard: schedule, daily goals, pending approvals, low-stock alerts, weigh sessions. Custom report builder with dynamic filters, column selection, and CSV/PDF export for ad-hoc data analysis.

09 QR & NFC Strategy

QR Codes

  • • Documents — version-pinned on publish, superseded redirect
  • • Assignments — scan at station to start job
  • • Parts — scan for spec sheet + stock
  • • Bins / Locations — scan for cycle count
  • • Labels — printed at weigh station for traceback
  • • URL: https://shop.example.com/s/{model}/{id}

NFC Tags

  • • Affixed to boxes and pallets during packing
  • • Scanned at weigh station to link item → box → shipment
  • • Handheld NFC reader for stacking/loading verification
  • • Tag UID stored in NfcTag model, polymorphic to any container
  • • Read/write: tag stores shipment number + timestamp for offline lookup

│

│ ├── Docker buildx (linux/amd64)

│
│ Production: Rocky Linux (bare metal or VM)
│ └── Incus / LXD system containers
│ ├── app Rails (Puma + Kamal proxy)
│ ├── worker Solid Queue
│ ├── postgres PostgreSQL 16
│ └── redis Redis (cache + queues)
│
│ Shop Floor Stations
│ ├── Weigh stations (browser + scale + camera + printer + NFC)
│ ├── Production station tablets (browser, QR scanning)
│ └── Handheld NFC readers (packing / stacking)
│
└─────────────────────────────────────────────────┘

Local Dev

Multipass spins up Ubuntu VMs on Apple Silicon. Matches the Rocky Linux production environment closely. Docker buildx cross-compiles amd64 images.

Deploy

Kamal deploys via SSH to Incus/LXD containers on Rocky Linux. App hot-swapped; Solid Queue workers drain gracefully. Incus is the community fork of LXD — either works.

Backup & Retention

pg_dump cron to S3. Weigh session photos stored on local disk at each station, reported to server. Auto-purge photos after 365 days. PaperTrail versions for point-in-time recovery. Inventory transactions append-only.

14 Milestones

M1

Auth + Roles + Doc CRUD

Rails new, Devise + Pundit, role seed data (Viewer, Operator, Author, Reviewer, Approver, Scheduler, Admin), Document CRUD with ActionText, Tailwind layout shell, categories.

M2

Document Workflow + Versioning

AASM state machine, submit/review/approve/publish/archive, PaperTrail, version history UI, side-by-side diffs, threaded comments.

M3

Production Scheduling + Daily Goals

Shift / WorkOrder / Assignment / WorkStation models, simple_calendar views, drag-and-drop, "My Schedule", DailyGoal model, real-time progress tracking per station and worker.

M4

Weigh Station & Packing

WeighStation / WeighSession models, scale integration, camera capture (Active Storage), label printing, 6-step station UX flow. NFC tag scanning for boxing/shipment tracking. Shipment model.

M5

Inventory + Warehouse Views

Part / StockLocation (with pos_x/pos_y) / InventoryTransaction / BOM models, stock views, transaction journal, low-stock alerts, 2D warehouse SVG map, location hierarchy browser.

M6

QR Codes + Search + Dashboard

QR generation across all modules, pg_search, unified Chartkick dashboard (goals, schedule, approvals, alerts, weigh log), batch QR export, audit log viewer.

M7

Integration + Deploy

Auto-deduct inventory on weigh session complete, NFC/QR end-to-end flow, Dockerfile, Kamal config, Multipass dev setup, Rocky Linux host, Incus/LXD containers, SSL, backup cron, smoke tests, go live.
